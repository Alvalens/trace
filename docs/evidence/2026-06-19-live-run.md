## Handover for the prose-night morning (step 4) ‚Äî key behaviors

- **Cross-format threading.** Prose updates land on the structured thread:
  - 112 aircon ‚Äî `ext_0002`+`evt_0002`
  - 309 deposit ‚Äî `ext_0006`+`evt_0007`
  - 312 no-show charged ‚Äî Chinese "settle ‰∫Ü" resolved the structured no-show (`ext_0004`+`evt_0010`)
  - 215 corridor leak ‚Äî prose update threaded onto the structured leak (`ext_0003`+`evt_0008`),
    one thread, not two.
- **Critical, action-first.** 208 jammed safe (passport + cash locked in, guest flying out) is
  Critical (`ext_0008`); 309 passport/booking-name mismatch is Critical (`evt_0006`).
- **Contradiction flag (205).** Structured shows the room in-house; prose observed it empty ‚Äî
  surfaced as `contradiction` ("verify before billing"), sources `evt_0024`+`ext_0007`, asserting
  neither side.
- **Grounding never silently drops.** If an extracted excerpt fails verbatim verification it is
  withheld from the buckets and raised as an `unverified` flag carrying its time/safety signals, so
  a critical line can't vanish. (In an earlier run the model mistyped one CJK character in the 208
  excerpt and it was dropped silently ‚Äî that is the failure this mechanism now prevents.)
